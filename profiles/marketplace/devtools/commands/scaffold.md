Scaffold a new module or component following the project's existing conventions.

Usage: /scaffold <type> <name>
Examples: /scaffold component UserCard, /scaffold api /users

1. Detect the project framework and file structure conventions
2. Create the file(s) matching existing naming and directory patterns
3. Include standard boilerplate (imports, exports, types) consistent with the codebase
4. Add a basic test file if the project has a test directory
